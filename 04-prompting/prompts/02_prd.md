# Phase 2: PRD Generation Prompts

```
Create a detailed Product Requirements Document (PRD) for my app.

App Concept: [3-5 SENTENCE DESCRIPTION]
Target User: [WHO — AGE, OCCUPATION, TECH LEVEL]
Platform: [iOS / Android / Web / Cross-platform]
Monetization: [Free / Freemium / Subscription / One-time]
MVP Timeline: [WEEKS AVAILABLE]

Generate with these sections:
1. Overview — One-line description, problem statement, target user
2. Goals & Metrics — Business goals with measurable targets
3. Features:
   - P0 (Must Have): 4-6 features with user stories and acceptance criteria
   - P1 (Should Have): 3-5 features for v1.1
   - P2 (Nice to Have): Future ideas
   - Out of Scope: What we're NOT building
4. User Scenarios — New user, returning user, power user walkthroughs
5. Technical Constraints — Offline, auth, storage, min OS
6. Launch Strategy — Scope, timeline, distribution

CRITICAL: Every feature needs a testable acceptance criterion. No vague descriptions.
```
